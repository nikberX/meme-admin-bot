# Meme Admin Bot

Telegram-бот для личного канала с мемами. Бот принимает:

- ссылки на ролики, которые умеет скачивать `yt-dlp` (например YouTube Shorts, Instagram Reels);
- фото и видео, отправленные напрямую боту;
- пересланные сообщения из Telegram, если медиа не защищено `protected content`.

После импорта бот создает черновик, дает возможность обновить подпись и источник, а затем публикует медиа в канал.

## Что умеет MVP

- импорт по URL через `yt-dlp`
- импорт фото и видео из Telegram
- импорт пересланных сообщений с медиа из Telegram
- хранение черновиков на диске
- ручная публикация в канал
- добавление подписи и источника перед публикацией

## Как работает сценарий

1. Вы отправляете боту ссылку, фото, видео или пересланное сообщение с медиа.
2. Бот скачивает файл и создает черновик.
3. Если черновик создан из URL, бот прикладывает скачанное медиа в ответе, чтобы его можно было заранее просмотреть или скачать.
4. Бот возвращает `draft_id`.
5. При необходимости вы задаете подпись и источник:

```text
/caption <draft_id> Смешной мем про...
/source <draft_id> https://instagram.com/...
```

6. Вы публикуете черновик:

```text
/publish <draft_id>
```

## Быстрый деплой на сервере через Docker

Это основной рекомендованный способ. Подойдет для Ubuntu VPS, Hetzner, DigitalOcean, любой Linux-машины с Docker.

### 1. Что должно быть на сервере

- установлен `docker`
- установлен `docker compose`
- открыт исходящий доступ в интернет
- достаточно места на диске для временных и сохраненных медиа

### 2. Клонируйте репозиторий

```bash
git clone https://github.com/nikberX/meme-admin-bot.git
cd meme-admin-bot
```

### 3. Подготовьте `.env`

```bash
cp .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=123456:telegram-bot-token
OWNER_USER_ID=
OWNER_USERNAME=telegram_username
OWNER_USERNAMES=telegram_username,another_username
CHANNEL_ID=@my_memes_channel
DATA_DIR=/app/data
TEMP_DIR=/app/data/tmp
YT_DLP_BINARY=yt-dlp
YT_DLP_EXTRA_ARGS=
YT_DLP_SLEEP_SECONDS=3
FFMPEG_BINARY=ffmpeg
```

### 4. Запустите контейнер

```bash
docker compose up -d --build
```

### 5. Проверьте логи

```bash
docker compose logs -f
```

Если все настроено верно, бот начнет принимать сообщения сразу после старта.

### 6. Обновление после изменений

```bash
git pull
docker compose up -d --build
```

## Полный гайд: создать и настроить Telegram-бота

### 1. Создайте бота через BotFather

1. Откройте Telegram.
2. Найдите `@BotFather`.
3. Отправьте команду `/newbot`.
4. Укажите имя бота.
5. Укажите username бота, он должен оканчиваться на `bot`.
6. BotFather вернет `BOT_TOKEN`.

Пример токена:

```text
123456789:AAExampleTokenFromBotFather
```

Именно это значение нужно записать в `BOT_TOKEN`.

### 2. Укажите владельца бота

Можно использовать один из трех вариантов:

- `OWNER_USER_ID` - numeric Telegram user id
- `OWNER_USERNAME` - Telegram username без `@`
- `OWNER_USERNAMES` - список Telegram username через запятую

Если не хотите сейчас искать numeric id, просто укажите ваш username.

Пример:

```env
OWNER_USERNAME=nikberX
```

Для нескольких владельцев:

```env
OWNER_USERNAMES=nikberX,nimperia
```

Если хотите использовать numeric id:

Это ваш личный Telegram user id. Бот будет принимать команды только от него.

Самый простой способ:

1. Найдите в Telegram любого бота вида `userinfobot`, `getmyid_bot` или аналогичный.
2. Отправьте ему любое сообщение.
3. Возьмите ваш numeric id из ответа.

Запишите его в `OWNER_USER_ID`.

### 3. Создайте или выберите канал для мемов

Если канала еще нет:

1. Создайте новый Telegram channel.
2. При желании задайте публичный username, например `@my_memes_channel`.

### 4. Добавьте бота администратором канала

Это обязательно. Иначе бот не сможет публиковать посты.

1. Откройте канал.
2. Перейдите в управление администраторами.
3. Добавьте туда вашего бота.
4. Дайте право публиковать сообщения.

### 5. Укажите `CHANNEL_ID`

Самый простой вариант:

- если у канала есть username, используйте его, например `@my_memes_channel`

Альтернатива:

- можно использовать numeric chat id канала, если он у вас уже есть

Для большинства случаев username канала достаточно.

### 6. Первый запуск и проверка

1. Запустите бота.
2. Откройте диалог с ботом.
3. Отправьте `/start`.
4. Затем отправьте ссылку или картинку.
5. Бот должен создать черновик и прислать инструкции с `draft_id`.

## Использование

### Команды

- `/help`
- `/list`
- `/caption <draft_id> <текст>`
- `/source <draft_id> <текст или ссылка>`
- `/publish <draft_id>`
- `/delete <draft_id>`

### Пример типичного сценария

1. Отправляете боту ссылку на Shorts или Reels.
2. Получаете ответ с `draft_id`.
3. При необходимости добавляете подпись:

```text
/caption m123456789 Мем про созвоны в пятницу
```

4. При необходимости добавляете источник:

```text
/source m123456789 https://youtube.com/shorts/abc
```

5. Публикуете:

```text
/publish m123456789
```

## Локальный запуск без Docker

Подходит для разработки или запуска на вашей машине.

### Зависимости

Нужны:

- Go 1.25+
- `yt-dlp`
- `ffmpeg`

### macOS

```bash
brew install yt-dlp ffmpeg
```

### Ubuntu/Debian

```bash
sudo apt update
sudo apt install -y ffmpeg python3 python3-pip
pip3 install --break-system-packages yt-dlp
```

### Запуск

```bash
cp .env.example .env
export $(grep -v '^#' .env | xargs)
go run .
```

## Структура деплоя

- `docker-compose.yml` поднимает один сервис бота
- `Dockerfile` собирает Go-бинарник и ставит `ffmpeg` + `yt-dlp`
- `./data` монтируется в контейнер и хранит:
- загруженные медиа
- `drafts.json`
- временные файлы

## Ограничения

- Если сообщение в Telegram нельзя переслать из-за защиты контента, бот тоже не сможет его скачать.
- Сейчас управление публикацией идет через команды, без inline-кнопок.
- Бот рассчитан на одного владельца через `OWNER_USER_ID`.
- Нет очереди отложенной публикации и автопостинга по расписанию.

## YouTube замечания

YouTube иногда rate-limit'ит серверный IP. Если бот отвечает ошибкой вида `This content isn't available, try again later`, это почти всегда ограничение со стороны YouTube, а не ошибка Telegram или Go-кода.

Что уже включено в проект:

- `yt-dlp` запускается с задержкой между запросами
- в контейнере установлен `nodejs` для JS runtime

Что помогает дополнительно:

- подождать 30-60 минут после rate limit
- сменить IP или использовать прокси

## Практические замечания

- `OWNER_USER_ID`, `OWNER_USERNAME` или `OWNER_USERNAMES` ограничивает управление ботом владельцами.
- `CHANNEL_ID` можно задать как `@channel_username` или numeric chat id.
- Бот использует long polling, webhook не нужен.
- Для URL-импорта доступность конкретной платформы зависит от того, поддерживает ли ее установленный `yt-dlp`.
